Section 05

Detailed Call Flow

What happens technically during a call

üìû User Clicks "Call"
Browser sends request to server with phone number and user ID. Server validates user has permission to make calls.
üîë Twilio Token Generated
Server creates a Twilio access token with the user's identity. Token allows browser to connect to Twilio Voice SDK.
üîå Voice SDK Connects
Browser initializes Twilio Device with token. WebRTC connection established for browser-based calling.
üì≤ Outbound Call Initiated
Twilio routes call through TwiML application. Call SID generated. Server notified via webhook. Database record created.
üéôÔ∏è Media Stream Starts
Audio from both parties streams to server via WebSocket. Server forwards audio to Deepgram for transcription.
üìù Live Transcription
Deepgram returns real-time text. Server broadcasts transcript to user's browser. Supervisor can also see it.
üí° AI Coaching
Every few transcript updates, Claude AI analyzes conversation. Coaching suggestions sent to rep's screen in real-time.
üî¥ Recording Saved
Twilio records call with dual-track audio. When call ends, recording URL sent to server via callback webhook.
üìä Call Completed
User logs outcome. AI generates summary. All data saved to database. Stats updated. Dashboard refreshes.

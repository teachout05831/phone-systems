## Database Schema (User & Calls)

```sql
-- Users Table
CREATE TABLE users (
  id            UUID PRIMARY KEY,
  email         VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name          VARCHAR(255) NOT NULL,
  role          ENUM('sales_rep', 'supervisor', 'admin') NOT NULL,
  team_id       UUID REFERENCES teams(id),
  phone_number  VARCHAR(20),           -- Their direct line if assigned
  twilio_identity VARCHAR(255),        -- For Twilio client identity
  is_active     BOOLEAN DEFAULT true,
  created_at    TIMESTAMP DEFAULT NOW(),
  updated_at    TIMESTAMP DEFAULT NOW()
);

-- Teams Table
CREATE TABLE teams (
  id            UUID PRIMARY KEY,
  name          VARCHAR(255) NOT NULL,
  supervisor_id UUID REFERENCES users(id),
  created_at    TIMESTAMP DEFAULT NOW()
);

-- Calls Table
CREATE TABLE calls (
  id            UUID PRIMARY KEY,
  call_sid      VARCHAR(255) UNIQUE NOT NULL,  -- Twilio's call SID
  user_id       UUID REFERENCES users(id),      -- WHO made/received
  team_id       UUID REFERENCES teams(id),

  direction     ENUM('inbound', 'outbound') NOT NULL,
  from_number   VARCHAR(20) NOT NULL,
  to_number     VARCHAR(20) NOT NULL,
  customer_name VARCHAR(255),

  status        ENUM('initiated', 'ringing', 'in-progress', 'completed',
                     'busy', 'no-answer', 'failed', 'missed') NOT NULL,
  outcome       VARCHAR(100),

  start_time    TIMESTAMP,
  end_time      TIMESTAMP,
  duration      INTEGER,                        -- seconds

  recording_url VARCHAR(500),
  recording_sid VARCHAR(255),

  has_transcript BOOLEAN DEFAULT false,

  notes         TEXT,

  created_at    TIMESTAMP DEFAULT NOW(),
  updated_at    TIMESTAMP DEFAULT NOW()
);

-- Call Transcripts Table
CREATE TABLE call_transcripts (
  id            UUID PRIMARY KEY,
  call_id       UUID REFERENCES calls(id),
  speaker       ENUM('rep', 'customer') NOT NULL,
  text          TEXT NOT NULL,
  timestamp     TIMESTAMP NOT NULL,
  is_final      BOOLEAN DEFAULT true,
  created_at    TIMESTAMP DEFAULT NOW()
);

-- AI Analysis Table
CREATE TABLE call_analysis (
  id            UUID PRIMARY KEY,
  call_id       UUID REFERENCES calls(id) UNIQUE,
  overview      TEXT,
  sentiment     ENUM('positive', 'neutral', 'negative'),
  action_items  JSONB,                          -- Array of strings
  coaching_tips JSONB,                          -- Tips shown during call
  created_at    TIMESTAMP DEFAULT NOW()
);
```

## Summary

This design enables:
- **Multiple users** logging in with their own credentials
- **Role-based views** showing appropriate data per role
- **Call attribution** - every call tied to a specific user
- **Team structure** - supervisors see their team's activity
- **Real-time monitoring** - supervisors can watch live calls
- **Shared infrastructure** - all users on same Twilio account but properly segmented

The key insight is that the Twilio integration itself doesn't change much - the same API calls work for all users. What changes is:
1. **Who can see what** (data filtering by role)
2. **Who gets notified** (WebSocket room assignments)
3. **What actions are available** (UI buttons based on permissions)

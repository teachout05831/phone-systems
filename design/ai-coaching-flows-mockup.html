2. Multi-Step Conversation Flow

Example: Price Objection Flow with branching based on customer response

1
Customer Says
"That's way too expensive. I can't afford that right now."
System Detects
Category: Price Objection
Keywords matched: "expensive", "afford"
Action: Start Price Objection Flow
2
Rep Says (Step 1 of 4)
"I completely understand. Budget is always an important consideration. Let me ask you this - if price wasn't a factor, would this solution solve the problem you're facing?"
Customer Response Branches
If "Yes, but..."
Customer agrees it would help → Move to Value Justification (Step 2A)
If "I guess so"
Customer is uncertain → Move to Needs Discovery (Step 2B)
If "No"
Customer says no → Exit to Needs Analysis flow
3
Rep Says (Step 2A - Value Justification)
"Perfect. So you see the value in what we offer. Let me share how our customers typically see a return on this investment within [timeframe]... Would it help if I showed you the payment options we have available?"
4
Rep Says (Step 3 - Close Attempt)
"Based on what you've shared, the [plan/option] would work best for you. Should we get you started with that today?"
Key Principle: Diagnose Before Responding
The system identifies the objection type (price, timing, authority, etc.) BEFORE providing a script. This ensures reps address the actual concern rather than using generic responses.

4. AI Prompting with Knowledge Base Context

How customer statement + KB context flows to AI for smarter suggestions

->

Context:
- KB: "Cleaning Services"
- Category: Competitor
- Recent transcript
- Category scripts
->
Output
AI Suggests
"Ask: What do you like best about them? This helps uncover switching triggers."
AI Context Enhancement
When no exact script matches, the AI receives: (1) the detected objection category, (2) KB-specific scripts for that category, (3) recent transcript context. This produces industry-relevant suggestions rather than generic advice.
